Stage 1

Local Corpus + Tokenizer

Place local text, code, JSON, JSONL, or parquet files under the corpus folder, then train the tokenizer from that local data only.

Stage 2

Base Model Training

This trains the underlying language model from the same local corpus. No benchmark downloads or cloud logging are used in this path.

Advanced Base Train Controls
Base checkpoint guidance will appear here after the dashboard loads.
Stage 3

Chat Fine-Tuning

This stage uses only local JSONL conversation files. Publish your identity file, draft extra conversations inside assistant_sandbox, and point SFT at those local files.

Advanced SFT Controls
SFT checkpoint guidance will appear here after the dashboard loads.
Stage 4

Benchmark + Smoke Test

Keep evaluation short while iterating. This local-only check measures BPB on your corpus and samples from the checkpoint, and the benchmark button keeps that config stable so results are easier to compare across runs.

This fork runs in local-only mode. Base eval here supports local checkpoint BPB and sampling, but not HuggingFace model paths or CORE bundle evaluation.
No benchmark history yet. Run a benchmark after a training pass to compare outcomes.
Stage 5

Chat RL

This launches the built-in reinforcement learning path on top of an SFT checkpoint. It is more advanced than SFT and usually makes sense only after you already have a usable SFT model.

Recommended order: tokenizer -> base train -> chat SFT -> chat eval -> chat RL. If you are new, treat RL as an optimization step, not the starting point.
RL checkpoint guidance will appear here after the dashboard loads.
Stage 6

Chat Eval

This runs the built-in chat-model evaluation tasks against SFT or RL checkpoints. Use it to measure real task behavior instead of relying only on casual chat impressions.

Available task names include `ARC-Easy`, `ARC-Challenge`, `MMLU`, `GSM8K`, `HumanEval`, and `SpellingBee`. Leave the task field blank to run the full built-in set.
